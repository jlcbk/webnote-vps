# 差异化改版计划

## 目标

把当前项目从“参考站同类实现”调整成更有辨识度的自部署临时中转工具。核心方向是：传送台首页、专业状态栏、明确的安全感和更清晰的分享/销毁动作。

## 设计原则

- 首页保留直接创建便签的效率，但视觉上改成“传送台控制台”。
- 编辑页优先服务重复使用，不做营销式装饰。
- 安全提示要诚实：强调 URL 隐蔽性、密码保护和 HTTPS，不暗示绝对安全。
- 只读链接、编辑链接、密码状态、有效期要在界面上更清楚。
- 不复制参考站的品牌、配色和文案。

## 开发阶段

### 1. 首页传送台

- 新增深色顶部控制区和大号中转面板。
- 首页输入时显示名称安全提示：空名称、简单名称、较安全名称。
- 保留随机创建、有效期预设和风险提示。

### 2. 编辑器专业化

- 顶部改为更明确的状态栏：便签名、保存状态、有效期、密码状态。
- 编辑区保留大面积纯文本输入，但减少参考站相似的视觉结构。
- 移动端保持侧栏可收起。

### 3. 安全侧栏

- 增加“保密状态”卡片，展示名称风险、密码状态、分享模式。
- 将只读链接和编辑链接做成更明确的分区。
- 删除动作改成“传输完成并销毁”，降低误解。

### 4. 验收

- `npm test` 全部通过。
- 首页和编辑页桌面/移动端截图无空白、无明显重叠。
- GitHub 上至少包含 baseline 提交和差异化改版提交。
